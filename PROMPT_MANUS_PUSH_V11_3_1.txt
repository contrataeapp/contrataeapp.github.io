Trabalhe APENAS no repositório existente abaixo. NÃO crie um repositório novo.

REPOSITÓRIO CORRETO:
https://github.com/contrataeapp/contrataeapp.github.io

OBJETIVO:
Subir a versão corrigida entregue no arquivo ZIP para o mesmo repositório que já está conectado ao Render.

PASSO A PASSO:

1. Abra/clonar o repositório existente:
   git clone https://github.com/contrataeapp/contrataeapp.github.io.git
   cd contrataeapp.github.io

2. Apague os arquivos locais do projeto atual, mantendo a pasta .git.

3. Extraia o conteúdo do ZIP desta entrega e copie tudo para dentro da pasta do repositório.
   Importante: sobrescrever os arquivos existentes com os novos arquivos.

4. Confirme que o remote continua sendo o mesmo:
   git remote -v
   Deve apontar para contrataeapp/contrataeapp.github.io

5. Revise rapidamente se os arquivos principais foram atualizados:
   - server.js
   - routes/dashboards.js
   - views/auth/login.ejs
   - views/auth/cadastro.ejs
   - views/auth/completar-perfil.ejs
   - views/dashboards/profissional-onboarding.ejs
   - views/dashboards/profissional-dashboard.ejs
   - views/auth/selecionar-tipo.ejs
   - public/js/admin.js

6. Faça o commit:
   git add .
   git commit -m "fix: v11.3.1 cadastro, validacoes, mobile e categorias admin"

7. Faça push para o repositório CORRETO:
   git push origin main

8. NÃO criar outro repositório.
   NÃO rodar git init.
   NÃO trocar o remote.
   NÃO publicar em contrataeapp-v11-3 ou qualquer outro repo.

RESULTADO ESPERADO:
- O commit deve aparecer em https://github.com/contrataeapp/contrataeapp.github.io
- O Render deve detectar automaticamente o push e fazer deploy da versão nova.
